type="VarName" - нужен для контроля типа данных в столбце. Сейчас есть 4 типа: 
1. VapName – допустимое в ST имя переменной, 
2. Number – любое число, 
3. Int – целое число, 
4. Bool – булевое значение, может быть ИСТИНА, ЛОЖЬ, 1, 0, true, false 
Если тип данных в ячейке не соответствует, будет сообщение об ошибке

default="3" - значение, которое пишется в ячейку БД, если ячейуа Excel пустая. Если ячейка в Excel пустая, а default не определён,
будет сообщение об ошибке

registr="down" - приведение строки взятой из ячейки Excel к определённому регистру. Может иметь значения down и up

replace="true" repFrom1="&gt;" repTo1="больше" - позволяет заменять определённые значения из ячейки Excel на другие. 
Если этот атрибут определён - конфигуратор ищет пары атрибутов repFrom1 и repTo1, repFrom2 и repTo2 и т.д.. 
Таких пар может быть сколько угодно, но номера должны идти подряд.
Далее конфигуратор проверяет не совпадает ли значение вычитанное из чейки Excel со значением одного из repFromN.
Если совпадает - заменяет на значение соответствующего repToN
Может применяться в комплекте с registr. В этом случае registr применяется раньше

swith="тип" var1="А" default1="авар." default="упр." - позволяет делать несколько дефаултов.
Если в ячейке Excel что-то есть - в БД пишется оно. Если  ячейка Excel пуста, конфигуратор проверяет, что записано в ячейке БД,
 указанной в swith
Далее ищется совпадение со значением атрибута varN выбирается дефаултное значение. 
Если ни один var не подошёл, в ячейку БД пишется значение default без номера

add1="типПостфикс" - позволяет добавлять к значению ячейки БД значения других ячеек. 
addText1=" " - позволяет добавлять к значению ячейки БД произвольный текст. 
Обе могут применять в комбинации с любыми другими атрибутами. Выполняются всегда последними

Общее замечание: если есть атрибут с числом в конце - надо искать другие атрибуты с 
последовательно увеличивающимися числами вконце
